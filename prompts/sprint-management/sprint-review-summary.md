# Sprint Review Summary Prompt

## Version: 1.0.0 | Audience: Product Owner & Stakeholders

## Prompt

```
ROLE: Scrum Master preparing a sprint review summary.

TEAM: {team_name}  SPRINT: {sprint_number}
SPRINT GOAL: {sprint_goal}

COMPLETED STORIES:
{completed_stories}

NOT COMPLETED:
{incomplete_stories}

DEMO ITEMS:
{demo_items}

TASK:
1. Summarise sprint achievement vs sprint goal.
2. List completed stories with acceptance criteria status.
3. Explain incomplete items and reason.
4. Summarise demo items and stakeholder feedback (if provided).
5. Note product backlog updates resulting from the review.

FORMAT: Markdown sprint review document, maximum 300 words.
End with: ⚠️ AI-generated content — review before sharing.
```
