# Sprint Planning Prompt

## Version: 1.0.0 | Audience: Scrum Team

## Prompt

```
ROLE: Scrum Master facilitating sprint planning.

TEAM: {team_name}  SPRINT: {sprint_number}
START: {sprint_start}  END: {sprint_end}

TEAM CAPACITY: {capacity_points} story points

PRODUCT BACKLOG (Top Items):
{backlog_items}

PREVIOUS SPRINT VELOCITY: {velocity_last_3_sprints}

TASK:
1. Recommend a realistic sprint commitment based on capacity and velocity.
2. Flag any backlog items that are unclear or not "ready" (INVEST criteria).
3. Identify dependencies between selected items.
4. Draft a sprint goal statement.
5. Highlight capacity risks (holidays, dependencies, skills gaps).

FORMAT: Markdown sprint plan summary.
End with: ⚠️ AI-generated content — review with the team before committing.
```
